# Graphus v0.0.1

**Version:** 0.0.1
**Date:** 2026-06-15
**Tag:** `v0.0.1`

## Summary

This is the **first tagged release** of **Graphus**, a Label Property Graph (LPG)
database server written in Rust and designed to operate under extreme load and
concurrency. The release packages two things adopters can use today:

1. The **single-node correctness core** — ACID transactions, a Cypher query engine
   targeting full openCypher TCK compliance, Bolt over Unix Domain Sockets and TCP,
   a Web REST API, authentication and RBAC, encryption at rest, and observability.
2. A **production-grade, multi-architecture Docker deployment** — a single container
   image that runs on `linux/amd64` and `linux/arm64` (Raspberry Pi 5 and Apple
   Silicon included), published automatically to the GitHub Container Registry on
   tagged releases.

Graphus uses a **multigraph** model by default and aims for byte-for-byte wire
interoperability with the Neo4j driver ecosystem via Bolt and PackStream.

## Highlights

- **ACID single-node core** — MVCC with Serializable Snapshot Isolation, an
  ARIES-style write-ahead log with group commit and checkpoints, and crash recovery.
- **Cypher** — parse → plan → execute pipeline targeting the pinned openCypher
  `2024.3` TCK.
- **Three connection interfaces** — Bolt over UDS, Bolt over TCP (`bolt://`, TLS),
  and a Web REST API with an OpenAPI document and health endpoints.
- **Security by default in the container** — first-boot self-signed TLS plus a random
  JWT secret, fine-grained RBAC, and AES-256-GCM encryption at rest.
- **Multi-architecture image** — one image for amd64 and arm64, published to GHCR with
  provenance and SBOM attestations.

## What is included

### Storage and transactions

- Record store with index-free adjacency, token store, and element-ID mapping.
- B+-tree, token-lookup, composite, and relationship-property indexes, plus
  constraints.
- MVCC + Serializable Snapshot Isolation transaction manager.
- ARIES-style write-ahead log: group commit, checkpoints, and crash recovery.
- AES-256-GCM encryption at rest for store pages, WAL frames, and backup envelopes,
  with crash-safe key rotation.

### Query and protocols

- Cypher engine targeting 100% openCypher TCK compliance (`2024.3`).
- Bolt 5.x with PackStream serialization over both UDS (IPC) and TCP. TCP is secured
  with TLS for the Neo4j driver ecosystem.
- Web REST API: transactional routes, OpenAPI document, and liveness/readiness probes.

### Security

- Peer-credential, JWT/Bearer authentication and fine-grained RBAC, shared across all
  listeners, backed by a durable, crash-safe security catalogue.

### Operations and tooling

- Built-in metrics, audit log, and admission control in the server process.
- Deterministic Simulation Testing (DST): a VOPR-style deterministic simulator with a
  scenario battery, fault injection, and Elle/Adya isolation checking.

### Deployment (new in this release)

- Production-grade `Dockerfile` and `docker-compose.yml`.
- Entrypoint that, on first boot, provisions a self-signed TLS certificate and a random
  JWT secret under `/data`, so Bolt and REST are encrypted out of the box. The server
  runs as the unprivileged `graphus` user (uid 10001); all durable state lives under
  `/data`.
- GitHub Actions workflow that builds `linux/amd64` and `linux/arm64` on every change
  and publishes a multi-architecture manifest to GHCR on `v*` tags.

## Getting started

```sh
# Build the image from this repository (native architecture):
docker build -t graphus:latest .

# Run it with a durable volume and both listeners published:
docker run -d --name graphus \
  -p 7687:7687 \           # Bolt over TCP (Neo4j drivers)
  -p 7474:7474 \           # Web REST API
  -v graphus-data:/data \  # databases persist here
  graphus:latest

# Liveness check (-k because the quickstart certificate is self-signed):
curl -k https://localhost:7474/health/live   # -> live
```

The local quickstart credentials are user `graphus` / password `graphus-local`.

## Security notes

The container quickstart ships **local-sandbox defaults only**: a **self-signed**
certificate (not CA-trusted) and a **well-known admin password**. They let you run
encrypted out of the box, but they are **not** suitable beyond a local sandbox. Before
any non-sandbox use, supply a CA-issued certificate (`GRAPHUS_TLS_CERT_PATH` /
`GRAPHUS_TLS_KEY_PATH`), a strong `[auth] admin_password`, and a real
`GRAPHUS_JWT_SECRET`. See the README "Production / TLS" section.

- **Operating systems:** Linux, macOS, and Raspberry Pi OS.
- **Architectures:** `x86_64` / `amd64` and `aarch64` / `arm64` (Apple Silicon and

  `bolt+ssc://`).
